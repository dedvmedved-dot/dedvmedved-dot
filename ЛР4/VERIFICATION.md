## Найденные проблемы

### 1. 🔴🔴🔴 CRITICAL: VIP конфликтует с Proxmox (БЛОКИРУЕТ)

Книга (стр. 75, 315, 396, 559, 641, 646, 650, 652, 664, 666, 671, 673, 689, 694, 713, 723, 744, 787):
```
VIP 192.168.0.200
```

| Адрес | Назначение |
|-------|-----------|
| 192.168.0.200 | Proxmox VE (управление) |
| 192.168.0.200 | VIP для балансировщиков (в книге) |

**Исправление**: использовать 192.168.0.210 (применено).

### 2. 🔴 CRITICAL: Отсутствует Terraform-код (Баг 0)

Книга (стр. 328): «В Terraform/OpenTofu нужно описать четыре ВМ» — только таблица параметров, кода нет.
Комплект (zip): только `mkdir -p terraform ansible`, main.tf отсутствует.

Студент вынужден:
- Либо писать с нуля (высокий порог входа)
- Либо копировать из ЛР1/ЛР3 (копирование багов: clone, student, local-lvm)
- Либо искать в комплекте (отсутствует)

### 3. 🔴 CRITICAL: ansible_user=student (сквозной Баг 2)

Книга (стр. 369-372, 376-381, 638, 640, 679-681, 683-685, 697-699, 701-703, 709-711, 717-719, 745-746):
- `ansible_user=student`
- `ssh student@192.168.0.201`

Cloud-образы Ubuntu создают пользователя `ubuntu`, не `student`.

### 4. 🔴 CRITICAL: Отсутствует clone (сквозной Баг 1)

Terraform-код не приведён, поэтому clone не упомянут. Но даже при самостоятельном написании студент не узнает о необходимости блока clone из книги.

### 5. 🟡 MEDIUM: datastore_id не параметризован (сквозной Баг 3)

В таблице параметров (стр. 330-359) нет поля datastore_id. На ZFS-стендах хранилище — vm-storage, не local-lvm.

### 6. 🟡 MEDIUM: cicustom не очищен (сквозной Баг 4)

Не упомянут. При клонировании шаблона cicustom может переопределить cloud-init.

### 7. 🟡 MEDIUM: ANSIBLE_HOST_KEY_CHECKING не упомянут (сквозной Баг 5)

Ни одна команда Ansible в книге не содержит ANSIBLE_HOST_KEY_CHECKING=False.

### 8. 🟡 MEDIUM: Интерфейс eth0 не универсален

Книга (стр. 398): `vip_interface: "eth0"`, сноска (стр. 413) признаёт проблему, но не даёт решение.

### 9. 🟡 MEDIUM: 4 ВМ в книге, 3 в стенде

Книга предполагает lb-01 + lb-02 + web-01 + web-02 (4 ВМ). В стенде lb-02 отсутствует (упрощённый деплой). Для controlled failure с переходом VIP нужен lb-02.

### 10. 🟡 MEDIUM: VM ID пересекаются с другими ЛР

Книга: 301, 302, 311, 312. В реальном деплое: 401, 411, 412.

### 11. 🟢 LOW: bash-синтаксис

`for i in {1..6}` (стр. 547, 652, 713, 787) — не работает в dash. Решение: `for i in $(seq 1 6)`.

### 12. 🟢 LOW: IP-адреса учебные

Книга: 192.168.0.101/102/201/202. Реальный деплой: 192.168.0.140/141/142.

2. inventory.ini — реальные IP, ansible_user=ubuntu
3. all.yml — VIP 192.168.0.210, eth0, реальные backend IP
